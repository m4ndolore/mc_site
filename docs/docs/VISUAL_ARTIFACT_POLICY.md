# Visual Artifact Storage Policy

## Categories

| Category | Location | Tracked in git? | Retention |
|----------|----------|-----------------|-----------|
| **Playwright baselines** | `tests/visual/screenshots/` | Yes | Permanent (auto-updated by CI) |
| **Ad-hoc MCP captures** | `.playwright-mcp/` | No (gitignored) | Local only — developer ephemeral |
| **CI failure artifacts** | GitHub Actions artifacts | No | 14 days (per `ci.yml`) |
| **Site assets** | `assets/` | Yes | Permanent (production images) |
| **Doc illustrations** | `docs/*/assets/` | Yes | Permanent (referenced in docs) |

## Rules

1. **Playwright baselines** are the only screenshots committed to git for regression testing. They live in `tests/visual/screenshots/` and are auto-generated by CI on Linux to avoid cross-platform font rendering differences.

2. **Ad-hoc verification screenshots** (Playwright MCP, manual captures) must NOT be committed. The `.playwright-mcp/` directory is gitignored. Any root-level `*.png` captures (e.g., `access-dark-full.png`, `builders-dark-theme.png`) are stale artifacts from past sessions and should be removed.

3. **Root-level PNGs are prohibited.** No `*.png` files should exist in the repository root. The gitignore blocks common patterns (`verification-*.png`, `homepage-header.png`, `opportunities-live.png`), but this is not exhaustive — don't commit ad-hoc screenshots anywhere outside `tests/visual/screenshots/` or `assets/`.

4. **CI failure diffs** are uploaded as GitHub Actions artifacts with 14-day retention. They are never committed to the repo.

5. **Doc screenshots** (`docs/screenshots/`, `docs/*/assets/`) are acceptable only when referenced by a living document. Orphan screenshots should be removed during RTMX hygiene passes.

## Gitignore Alignment

The `.gitignore` enforces this policy with:

```
# Ad-hoc Playwright MCP captures
.playwright-mcp/

# CI test outputs
tests/visual/test-results/
tests/visual/playwright-report/

# Root-level ad-hoc captures
*.png
!assets/**/*.png
!tests/visual/screenshots/**/*.png
!docs/**/*.png
!content/**/*.png
!styles/**/*.png
!opportunities/**/*.png
```

## Cleanup Procedure

When stale screenshots are found in git:

```bash
# Find tracked PNGs outside allowed directories
git ls-files '*.png' | grep -v -E '^(assets/|tests/visual/screenshots/|docs/|content/|styles/|opportunities/)'

# Remove and commit
git rm <stale-files>
git commit -m "chore: remove stale ad-hoc screenshots per VISUAL_ARTIFACT_POLICY"
```
