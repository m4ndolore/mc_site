# REQ-UX-016 — Consolidate Navigation to 5-6 Items

**Category:** UX / NAVIGATION  
**Priority:** MEDIUM  
**Phase:** 12  
**Status:** BACKLOG  

## Problem

Too many navigation items (Programs, Builders, Signals, Knowledge, Learn, Start, About) and too many competing CTAs. User doesn't know what to do first. The homepage CTA goes to /start, some buttons go to /access, some to /builders, some to /programs/the-combine.

## Acceptance Criteria

1. Primary nav reduced to 5-6 items maximum
2. Single clear primary funnel path from homepage
3. Secondary items consolidated into dropdowns or footer
4. Every page has one primary CTA that's consistent with the funnel

## Proposed Structure (to validate)

- Programs (dropdown: Combine, Residency, Launch triage)
- Builders (the directory/marketplace)
- Signals (content/analysis)
- Learn (toolkit/knowledge — consolidate /knowledge and /learn)
- About
- [CTA button: Get Access]

## Evidence (Critical Assessment 2026-05-16)

"You have 9 navigation items and I still don't know what to do first."
