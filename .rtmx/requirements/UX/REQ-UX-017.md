# REQ-UX-017 — Single Primary Funnel Path

**Category:** UX / CONVERSION  
**Priority:** MEDIUM  
**Phase:** 12  
**Status:** BACKLOG  

## Problem

Too many CTAs competing for attention. Homepage has: "Get Started" → /start, "Learn More" → /about, "Join Defense Builders" → /start, "Build with us" → /start, "Browse our Defense Builder partner network" → /builders, plus nav items. No clear single path.

## Acceptance Criteria

1. Homepage has ONE primary CTA that's visually dominant
2. Secondary CTAs are clearly subordinate (ghost/text style)
3. The primary funnel is: Homepage → /launch (triage) OR /access → Guild
4. All "Join" and "Get Started" buttons point to the same destination
5. Measure: user should understand what to click within 3 seconds of landing

## Notes

Ties to REQ-UX-016 (nav consolidation). The funnel simplification should happen alongside nav cleanup.
